# DESIGN.md Inspiration Index

Use this reference only when the user asks for a visual direction, moodboard, brand-like inspiration, or a `DESIGN.md` based on examples.

If a DESIGN.md catalog is available, inspect only the requested company/style entry. Otherwise use a live hosted reference only when current research is appropriate.

## Style Families

| Direction | Good reference names | Use when |
|---|---|---|
| Monochrome developer precision | `vercel`, `hashicorp`, `replicate` | infra, docs, developer tools, launch pages |
| Calm dense product UI | `linear.app`, `notion`, `superhuman` | dashboards, task tools, productivity flows |
| Technical trust | `mongodb`, `clickhouse`, `sentry`, `supabase` | databases, observability, backend platforms |
| Polished infrastructure marketing | `stripe`, `resend`, `coinbase` | payments, APIs, conversion pages |
| Premium product restraint | `apple`, `bmw`, `tesla` | product reveal, luxury, hardware-like UI |
| Playful devtool energy | `posthog`, `figma`, `lovable` | friendly SaaS, creative tools, onboarding |
| Cinematic dark | `runwayml`, `elevenlabs`, `spacex` | media, AI generation, immersive landing pages |
| Editorial/content-first | `wired`, `theverge`, `claude` | writing, knowledge, publishing, narrative pages |

## Selection Rule

Choose at most one primary reference and one secondary accent. If more are tempting, synthesize them into a project-specific direction before writing `DESIGN.md`.

## Output Rule

Never paste a company `DESIGN.md` into project memory. Distill the durable visual rules into the project's own `DESIGN.md` using `design-md-authoring.md`.
